Rownd will call this API after an authenticated session is established. This can happen directly after a sign-in, or when the page is reloaded with a currently authenticated user.

<Expandable title="Parameters" defaultOpen="true">
  <ResponseField name="url" type="string">
  The URL that Rownd will call
  </ResponseField>
  <ResponseField name="method" type="string">
  The HTTP method that Rownd will use. Can be either `"post"`, `"put"`, or `"get"`.
  </ResponseField>
  <ResponseField name="extra_headers" type="object">
  A map of key-value pairs that Rownd will include in the HTTP headers
  </ResponseField>
  <ResponseFIeld name="timeout" type="number">
  The HTTP request timeout in milliseconds. If not spescified, it is set to 10000 (10 seconds).
  </ResponseFIeld>
</Expandable>

If you specify a `"put"` or `"post"` method, the access token will be included in the request body as a field named `access_token`. For an HTTP `"get"`, it is included in the `Authorization` header.

There are two properties that Rownd looks for in the response body that drive behavior when thhe API call completes.

<Expandable title="Response Properties" defaultOpen="true">
  <ResponseField name="return_to">
  A relative path or URL to which Rownd will redirect
  </ResponseField>
  <ResponseField name="should_refresh_page">
  If `true`, Rownd will refresh the current page
  </ResponseField>
</Expandable>
