These docs shows how to add the **CNAME** and **TXT** records required to configure a custom authentication domain (e.g., for magic‑link emails, smart links, etc.) with Rownd. Each example uses:

- **CNAME**: points `<YOUR_CUSTOM_DOMAIN>` to your Rownd subdomain (`<ROWND_SUBDOMAIN>`).
- **TXT**: creates a verification record at `<TXT_RECORD_HOSTNAME>` with the token `<VERIFICATION_TOKEN>`.

Click the links for your DNS provider’s official docs for more details.

## 1. Cloudflare

- **Official Docs**: [Manage DNS Records – Cloudflare](https://developers.cloudflare.com/dns/manage-dns-records/)

### Add a CNAME Record

| Field  | Value                          |
|--------|--------------------------------|
| Type   | CNAME                          |
| Name   | `<YOUR_CUSTOM_DOMAIN>`         |
| Target | `<ROWND_SUBDOMAIN>`            |
| TTL    | Auto (or your preferred value) |

### Add a TXT Record

| Field   | Value                         |
|---------|-------------------------------|
| Type    | TXT                           |
| Name    | `<TXT_RECORD_HOSTNAME>`       |
| Content | `<VERIFICATION_TOKEN>`        |
| TTL     | Auto (or your preferred value)|

## 3. Google Cloud DNS

- **Official Docs**: [Resource Record Types – Cloud DNS](https://cloud.google.com/dns/docs/records)

### Add a CNAME Record

| Field                | Value                      |
|----------------------|----------------------------|
| DNS Name             | `<YOUR_CUSTOM_DOMAIN>.`    |
| Resource Record Type | CNAME                      |
| Canonical name       | `<ROWND_SUBDOMAIN>.`       |
| TTL                  | 300 sec (or your preference)|

### Add a TXT Record

| Field                | Value                          |
|----------------------|--------------------------------|
| DNS Name             | `<TXT_RECORD_HOSTNAME>`        |
| Resource Record Type | TXT                            |
| TTL                  | 300 sec (or your preference)   |
| TXT data             | `"<VERIFICATION_TOKEN>"`       |
